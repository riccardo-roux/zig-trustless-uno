A simple 2 players PQ trustless Uno built in Zig, using MLDSA87 and Kyber1024.

Here is how it works :
* users choose a server
* to make it easier, users do not exchange Dilithium/Kyber public keys (via a secure channel), but only `sha3_256(Dilithium_pk || Kyber_pk)` (and they just have to communicate to fetch it, with no way for someone to send another pubkey with the hash being the same)
* before the party begins, both users safely exchange the right pubkeys (they ask the server for the full pubkey, from its hash)
* from that point, everything is signed using Dilithium (the message contains the hash of the author's pubkey, and the full signature) and then encrypted using a random 256bit key (encapsulated for the other player's Kyber pk during the client-to-client handshake)
* to choose in which order the players play : each player generates a random 256bit key and hash it, both exchange their hash, then both reveal the values, and both compute the XOR of both plaintext values and the order is just determined by `sha3_256('the final XORed value' || player_id)`, with player_id being the hashed_pubkey. 
* to make a player pick a random card (at the beginning of the game, or each time a player has to draw cards) : both generate a random 256bit value and exchange their hashes, then everyone (except the player that is picking the card) reveal their value, and the player that was picking the card XOR all the plaintext values (including its own), and all player can just XOR all the plaintext values they have (so, not from the player that picks the card), and store it with the player hash (this way they know that the player has this card, and they can verify the player has it when they play it and therefore reveal the unhashed value), and the card is just the `(final XORed value) % (deck size)`.
